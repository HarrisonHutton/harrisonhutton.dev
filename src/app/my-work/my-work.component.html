My Work

Moog Inc.

(Internship: Summer '23)

At Moog, I worked on an R&D team with a goal of creating a self-driving, electric Packmule vehicle for use in FOD (Foreign Object Debris) detection on airfields. I learned how to use Rviz and Gazebo to visualize and simulate the vehicle and its sensors. Building out the Xacro description of the Packmule was a great introduction to ROS and URDF files. These were designed to interact directly with open source ROS code for Ackermann steering vehicles. After the driving mechanics were simulated successfully, I configured the LIDAR and camera sensors to communicate with the code for object detection.

M&T Universal Search

(Internship: Summer '22)

On an Agile team, I built the frontend of a universal document search tool during my Summer '22 internship. I learned how to use Angular, TypeScript, and SCSS to design an inviting and functional UI and communicate with a Java Springboot backend. This journey spans over a year of dedicated efforts, showcasing my proficiency in Angular, Firebase services (Auth, Firestore, Hosting), and effective leadership in delivering a tailored SaaS solution to meet the unique needs of GOMAD.

Bytecode Interpreter

As a semester-long project for my Programming Language's course, I built a bytecode interpreter from scratch in OCaml. I learned how to use a functional language to construct an abstract syntax tree for a given input file and then execute the file using a stack-based interpreter.


My goal throughout the project was to keep my code as modular as possible to isolate the individual working parts of the interpreter. I did this by separating the project into Types, Utilities, and Execution modules. This allowed me to quickly find and squash any bugs that appeared.


The toughest part of this project was implementing first-class functions and closures, but it did get done! Overall, this project was a great introduction to functional programming, pattern matching, and the power of recursion.

"Risk" Dice Roll Chances

For this project, I used Python to estimate the average win rate percentages for dice-rolling battles in the board game Risk through the use of Monte Carlo methods. The program was written from scratch to simulate a typical dice-rolling battle in Risk and determine the outcome based on the set of rolled numbers for each player.


The results were visualized using Excel and built using approximately 1500 'battles.' It's important to note that the results are approximate and should be interpreted as such. The outcome of each battle is determined by the set of rolled numbers for each player and the number of troops they have on the board.


This project provided an opportunity to apply my skills in Python and Monte Carlo methods to solve a problem and gain insights into the gameplay of Risk. The results were visualized using Excel to better understand the average win rate percentages for dice-rolling battles in the game.
